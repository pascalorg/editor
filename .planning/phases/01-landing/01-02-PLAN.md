  <action>
    Create a new file `apps/editor/app/opengraph-image.tsx` that uses Next.js's file-convention to auto-generate the OG image for the landing page (`/`).

    CRITICAL constraints (from research):
    - Use `import { ImageResponse } from 'next/og'` (bundled with Next.js 16 — do NOT install @vercel/og)
    - Use INLINE `style` props ONLY — Satori silently ignores Tailwind className. Every visual property must be in `style={{ ... }}`
    - Export the four required constants: `alt`, `size`, `contentType`, and a default async function returning ImageResponse
    - No `export const runtime` needed (defaults work)
    - Do NOT use external fonts in v1 — let Satori fall back to its default; we can layer in a custom font in a follow-up phase

    Required exports:
    ```ts
    export const alt = 'Archly — Collaborative 3D Building Design'
    export const size = { width: 1200, height: 630 }
    export const contentType = 'image/png'

    export default async function Image() {
      return new ImageResponse(
        (
          <div style={{ /* inline styles only — NO Tailwind classNames */ }}>
            ...
          </div>
        ),
        { ...size }
      )
    }
    ```

    Visual design (1200x630, dark theme to match landing):
    - Full-bleed background: linear-gradient(135deg, #0a0a0a 0%, #1a1530 50%, #2d1b4e 100%) — matches the indigo/violet brand
    - Centered flex column container, padding 80px
    - Top row: brand mark "Archly" in white, fontSize 48, fontWeight 800, letterSpacing -1
    - Headline: "Collaborative 3D Building Design" — fontSize 96, fontWeight 800, color white, lineHeight 1.05, letterSpacing -2, marginTop 40
    - Subhead: "Where teams build in 3D. WebGPU performance. Real-time collaboration." — fontSize 32, color #a1a1aa (zinc-400), marginTop 24, maxWidth 1000
    - Bottom row (flex row, gap 24): three pill badges with backgrounds #4f46e5 / #7c3aed / #a78bfa, white text, padding 12px 24px, borderRadius 999, fontSize 20, fontWeight 600 — labels: "WebGPU", "Real-time", "Open"
    - Every nested element MUST have display: 'flex' if it has multiple children (Satori requires this)

    Reminder: Next.js 16 — if you use a dynamic segment, `params` must be awaited. This route has no params, so no async params concern.

    Verify the file is recognized: after build, check `.next/server/app/opengraph-image` route is generated and `view-source:http://localhost:3000/` contains `<meta property="og:image"`.
