  <action>
Modify `apps/editor/lib/auth.ts`:

1. Import `GoogleProvider` from `next-auth/providers/google`.
2. Append to `providers` array (keep existing CredentialsProvider untouched):
   ```ts
   GoogleProvider({
     clientId: process.env.GOOGLE_CLIENT_ID!,
     clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
     allowDangerousEmailAccountLinking: true,
   })
   ```
   Why `allowDangerousEmailAccountLinking: true`: without it, a user who signed up with email/password and then tries Google sign-in with the same email gets silently blocked. Safe for Google because Google verifies email ownership.

3. Add a `signIn` callback to `callbacks` (BEFORE the existing `jwt`/`session` callbacks). This is required because we deliberately do NOT use `@auth/prisma-adapter` (it would force a DB session strategy and break our existing JWT setup). Manually upsert the User row so the JWT callback finds an `id`:
   ```ts
   async signIn({ user, account, profile }) {
     if (account?.provider === "google") {
       if (!user.email) return false;
       const email = user.email.toLowerCase();
       const dbUser = await prisma.user.upsert({
         where: { email },
         update: {
           name: user.name ?? undefined,
           image: user.image ?? undefined,
         },
         create: {
           email,
           name: user.name ?? null,
           image: user.image ?? null,
           // password intentionally null — OAuth-only account
         },
       });
       // Mutate user.id so the jwt callback picks up the DB id, not Google's sub
       user.id = dbUser.id;
     }
     return true;
   },
   ```

4. Verify the existing `jwt` callback already stores `token.id = user.id` — it does, leave as-is. This guarantees AUTH-04 session persistence continues to work for Google users.

5. Do NOT install `@auth/prisma-adapter`. Do NOT change `session.strategy` (must stay `"jwt"`).

Modify `apps/editor/env.mjs`:

6. Tighten the Google env vars from `.optional()` to required so the app fails fast if misconfigured:
   ```ts
   GOOGLE_CLIENT_ID: z.string().min(1),
   GOOGLE_CLIENT_SECRET: z.string().min(1),
   ```
   Leave `RESEND_API_KEY` optional (no email provider for v1 per locked decision).

7. Verify `NEXTAUTH_SECRET` and `NEXTAUTH_URL` are referenced (NextAuth v4 reads them automatically from `process.env` — no schema entry strictly required, but add `NEXTAUTH_SECRET: z.string().min(1)` to the server schema and `runtimeEnv` for fail-fast safety).
