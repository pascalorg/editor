**Part B — Modify `apps/editor/lib/auth.ts`:**

4. Import `GoogleProvider` from `next-auth/providers/google`.

5. Append to `providers` array (keep existing CredentialsProvider untouched):
   ```ts
   GoogleProvider({
     clientId: process.env.GOOGLE_CLIENT_ID!,
     clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
     allowDangerousEmailAccountLinking: true,
   })
   ```
   Why `allowDangerousEmailAccountLinking: true`: without it, a user who signed up with email/password and then tries Google sign-in with the same email gets silently blocked. Safe for Google because Google verifies email ownership.

6. Add a `signIn` callback to `callbacks` (BEFORE the existing `jwt`/`session` callbacks). This is required because we deliberately do NOT use `@auth/prisma-adapter` (it would force a DB session strategy and break our existing JWT setup). Manually upsert the User row using **email as the unique key** so existing credential users are linked, not duplicated:

   ```ts
   async signIn({ user, account, profile }) {
     if (account?.provider === "google") {
       if (!user.email) return false;
       const email = user.email.toLowerCase();
       const dbUser = await prisma.user.upsert({
         where: { email },
         update: {
           name: user.name ?? undefined,
           image: user.image ?? undefined,
         },
         create: {
           email,
           name: user.name ?? null,
           image: user.image ?? null,
           // password intentionally null — OAuth-only account
         },
       });
       // Mutate user.id so the jwt callback picks up the DB id, not Google's sub
       user.id = dbUser.id;
     }
     return true;
   },
   ```

   The `where: { email }` clause MUST be the unique key (Prisma's `User.email` is `@unique` per the existing schema). Do NOT use `userId` or `id` as the upsert key — that would create duplicates for existing credential users.

7. Verify the existing `jwt` callback already stores `token.id = user.id` — it does, leave as-is. This guarantees AUTH-04 session persistence continues to work for Google users.

8. Do NOT install `@auth/prisma-adapter`. Do NOT change `session.strategy` (must stay `"jwt"`).
